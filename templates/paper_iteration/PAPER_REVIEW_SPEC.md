# Paper Review Spec — {{PAPER_TITLE}}

## 审查流程

1. **提交审查请求**：提供摘要、结构、关键论点（不含全文）
2. **GPT 审查**：基于摘要和结构给出方向性反馈
3. **记录结果**：verdict + 反馈方向 + 建议修改点
4. **修订**：根据反馈修改草稿
5. **重新提交**：最多 3 轮审查

## 审查维度

| 维度 | 说明 |
|------|------|
| 结构 | 逻辑流、章节组织 |
| 论证 | 论点清晰度、证据链 |
| 创新性 | 与现有工作的差异 |
| 可读性 | 语言清晰度、术语一致性 |

## 审查结果格式

```yaml
review_round: 1
verdict: accepted | revision_required | rejected
feedback_summary: "方向性反馈摘要（不含全文引用）"
suggested_changes:
  - "建议 1"
  - "建议 2"
```

## 限制

- 审查请求不包含完整论文正文
- 不在仓库中存储 GPT 完整回复
- 最多 3 轮审查
